# Шпаргалка по файловому менеджеру Yazi

**Yazi** — это терминальный файловый менеджер, написанный на Rust с оглядкой на Vim. Идеально вписывается в рабочий процесс Neovim + zsh (vi-mode).

> [!ABSTRACT] Основная концепция
> Интерфейс разделен на три панели:
> 1.  **Слева:** Родительские каталоги (дерево навигации).
> 2.  **В центре:** Содержимое текущего каталога (файлы и папки).
> 3.  **Справа:** Панель предпросмотра (код, изображения, мета-информация).

---

## I. Навигация (в стиле Vim)

| Сочетание | Действие | Примечание |
| :--- | :--- | :--- |
| `j` / `k` | Перемещение вниз / вверх | Абсолютно как в Vim |
| `l` | **Войти** в каталог или **открыть** файл в `$EDITOR` | Важнейшая команда |
| `h` | **Выйти** в родительский каталог | |
| `gg` | Перейти в самый верх списка | |
| `G` | Перейти в самый низ списка | |
| `Ctrl + f` | Прокрутка на страницу вперед | |
| `Ctrl + b` | Прокрутка на страницу назад | |
| `H` / `L` | Переход по истории (назад / вперед) | Как `<C-o>` / `<C-i>` в Vim |

## II. Операции с файлами и каталогами

### Создание (⚠️ Изменения в v26+)

| Сочетание | Действие | Мнемоника / Нюанс |
| :--- | :--- | :--- |
| `a` | Создать файл **или папку** | `a`dd. Если имя заканчивается на `/` → создаст **папку** |
| `A` (Shift + a) | Создать папку (legacy) | ⚠️ Может не работать в некоторых терминалах |

> [!TIP] Универсальный паттерн создания
> Используйте `a` + `foldername/` для создания папок.
> Это работает стабильнее, чем `Shift+A`, и не зависит от настроек терминала.
>
> **Пример:**
> - `a` → `notes.txt` → создаст **файл**
> - `a` → `project/` → создаст **папку**

> [!EXAMPLE] Если Shift+A не работает
> Добавьте в `~/.config/yazi/keymap.toml`:
> ```toml
> [manager]
> keymap = [
>   { on = "A", run = "mkdir", desc = "Create directory" },
> ]
> ```

### Копирование, вставка, перемещение и удаление
| Сочетание       | Действие                  | Примечание                                                             |
| --------------- | ------------------------- | ---------------------------------------------------------------------- |
| `y`             | Копировать (yank) файлы   | Копирует пути для последующей вставки                                  |
| `x`             | Вырезать (cut)            | Готовит файлы к перемещению                                            |
| `d`             | Удалить в корзину (trash) | Отправляет в trash, не вырезает                                        |
| `p`             | Вставить (paste)          | Копирует или перемещает файлы в текущую папку                          |
| `D` (Shift + d) | Удалить навсегда          | ⚠️ remove --permanently: минуя корзину, без возможности восстановления |
| `r`             | Переименовать (rename)    | Поддерживает массовое переименование                                   |
> [!WARNING] Критически важно
> - **`d` (строчная)** = по умолчанию перемещает в корзину (безопасно) 🗑️
> - **`D` (заглавная)** = удаляет навсегда мимо корзины (опасно) ☠️
>
> Всегда проверяйте, какую клавишу нажимаете!

> [!TIP] Настройка под себя
> Если вы не пользуетесь корзиной и хотите, чтобы `d` удалял навсегда:
> ```toml
> # ~/.config/yazi/keymap.toml
> [mgr]
> prepend_keymap = [
>   { on = "d", run = "remove --force --permanently", desc = "Delete permanently" },
> ]
> ```
> После этого `d` будет работать как `rm -f`.
### Визуальный режим (массовые операции)

| Сочетание | Действие | Workflow |
| :--- | :--- | :--- |
| `v` | Визуальный режим | Выделите первый файл, двигайтесь `j`/`k` для выбора диапазона |
| `Ctrl + a` | Выделить **все** файлы | В текущем каталоге |
| `Space` | Точечное выделение | Переключить выделение на текущем файле |
| `Esc` | Снять все выделения | Выход из режима выделения |

> [!NOTE] Порядок действий
> 1. Выделите файлы (`v`, `Ctrl+a` или `Space`)
> 2. Нажмите `y` (копировать), `d` (вырезать) или `D` (удалить)
> 3. Перейдите в нужную папку и нажмите `p` (вставить)

## III. Поиск, сортировка и табы

| Сочетание | Действие | Примечание |
| :--- | :--- | :--- |
| `/` | **Поиск / Фильтр** | Динамическая фильтрация списка по имени |
| `s` | Меню **сортировки** | По имени, дате, размеру, типу |
| `gt` | Следующая вкладка | Как в Vim/Neovim |
| `gT` | Предыдущая вкладка | |
| `c` | Закрыть вкладку | `c`lose |
| ` ` (Пробел)| Новая вкладка | |
| `:` | **Командная строка** | Ввод команд: `mkdir`, `shell`, `help`, `set` |

## IV. Интеграция с Neovim

> [!SUCCESS] Бесшовный workflow
> Настройте Yazi так, чтобы по нажатию `l` он открывал файлы в уже запущенном экземпляре Neovim.
>
> **Рекомендуемый плагин:** [yazi.nvim](https://github.com/mikavilpas/yazi.nvim)
>
> Это создает опыт, когда вы вызываете Yazi из Neovim, находите файл, и он открывается как новый буфер в том же окне.

## V. Главный совет: Панель предпросмотра

Используйте правую панель на полную мощность. Перемещаясь с помощью `j` и `k`, вы можете:
*   📄 Мгновенно читать содержимое текстовых файлов (`.md`, `.txt`, `.php`, `.js`).
*   🖼️ **Просматривать изображения прямо в терминале** (требуется терминал с поддержкой протоколов графики: Kitty, WezTerm, iTerm2).
*   📊 Видеть структуру JSON и мета-данные файлов.
*   📁 Видеть содержимое архивов и каталогов без входа в них.

Это избавляет от необходимости открывать десятки файлов, чтобы найти нужный.
